# inventory

Инвентарь содержит обычные слоты для предметов, доступные игроку и заблокированные слоты. Он расширяемый и имеет лимит.

Необходимо добавить:
- [] перетаскивание предметов внутри инвентаря и в слоты экипировки + по кнопке "применить"
- [] соединение одинаковых предметов в стак (перетаскивание), если предметы не превышают лимит стака
- [] надевание эквипа даблкликом + по кнопке "применить"
- [] снятие экипировки\оружия\еды и пр. даблкликом и перетаскиванием
- [] перемещение еды и зелий в слот для еды даблкликом и перетаскиванием

Исправить или заменить:
- [] отображение подсказок о предметах при удержании пальца на слоте (убрать текущую логику с окошком над слотами). Сейчас в некоторых местах используется скрипт RewardItemHandler+ItemInfoManager, но там проблемы с позиционированием(вылезает за экран канваса). В инвентаре эти скрипты не используются, так как там есть ItemDescriptionUI, который как раз надо убрать. В общем, нужен отдельный скрипт, чтобы при нажатии на слот с предметом появлялась модалка с информацией.
- [] сейчас, если в слоте с оружием уже лежит предмет, то при выборе другого меча и нажатии на кнопку применить предыдущее оружие встаёт в пустой слот, а не на место нового. И еще, если в инвентаре нет места, в этой же ситуации старое оружие просто пропадает. Надо учитывать лимит инвентаря.
- [] можно совсем переписать или доработать текущую логику надевания оружия\экипировки на персонажа (сейчас инвентарь использует скрипт WeaponDisplay).

> [!NOTE]  
> В инвентаре (Inventory.cs) есть закомментированные строки связанные с квестами, типа QuestManager\CanQuestBeCompleted, или GameBookUI\CraftingUI в InventoryUI, прошу не удалять их.
